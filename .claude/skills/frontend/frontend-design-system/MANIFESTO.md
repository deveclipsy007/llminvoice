# MANIFESTO - frontend-design-system

## Proposito

Eliminar divergencia visual entre os fluxos do produto e evitar replicar interfaces rejeitadas.

## Limites

- Nao altera regras de negocio do admin.
- Nao introduz frameworks extras de UI sem necessidade.

## Riscos

- Mudancas visuais podem reduzir contraste se nao validadas.
- Ajustes de layout podem afetar componentes em paginas longas.

## Responsabilidade

- Produto define direcao estetica final.
- Engenharia implementa e valida responsividade.
